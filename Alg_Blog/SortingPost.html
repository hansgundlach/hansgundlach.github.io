Sorting Intro

Binary Tree Sorting

Sorting algorithms are one of the first topics students learn in computer science, Yet the theory behind sorting is large and there are tens if not hundreds of modern sorting algorithms

However a large variety of modern sorting algorithms realy on data structrures called trees. One of the most important tree data structures is the binary search tree. The binary search tree is built on the principle of binary search. Binary search of an array would involve comparing an element at the middle of an array to a selected element andif the middle element is less than desired check the middle element of the upper half of the sorted array. Binary search trees work in a very similar way. To find an element of a binary search tree compare the element your searching for with your currrent node. If your element is less than the current node go to the left node and vice versa if it si greater.

Radix Sort

So far all the sorting algoroithms I've shown sort in \$ O(n lgn) \$ time, However depending on the type of data we can do better. Radix sort integers in linear time . Radix sort does this by grouping integers based on significant digits. More specificaly radix sort running time is proportional to n the number of numbers and b the base which the numbers are in. Radix sort can be grouped in two parts. The first part of the algorithms looks at each significant digit and sorts by each digit. The second part namley the counting sort subroutines actually sorts numbers based on their digits. The fine details of radix sort are a bit complicated so I'm presenting a video below.

Radix sort is easy for integers but is hard to generalize and is therefore not a widley used sorting algorithmn despite is improved O(n) complexity.
