Hashing Algorithmns

A hash function is just a function that maps from keys to integers in constant time. Practicly, hashing can be done by using the memory adress of each peice of data being hashed as the hash value. There are 3 main beginner hasing algorithms. The first algorithmn is called the division algorithmn simply takes the remainder of a number divided by a prime power close to The second commoly used method is called the multiplication method. This method uses the function \$ h(k) = [(a*k) mod 2^w] >> (w-r) \$ where a is an odd integer from \$ 2^(w-1) \$ to \$ 2^w \$ .

Open Adressing

Open adressing is a method of hashing which does not need pointers. Open addressing puts all the hashes in a large array.Open adressing gets around collisions by moving forward or probing unfilled adresses afterward to put the hash. This has a few advantages and disadvantages,. The advantage is that hashing no longer need to store hash collisions and search through long lists. The disadvantage is the hash table involves longer run time to probe for where the hash value is in the table. Depending on the Haashing strategy there can also be disadvantages with clustering inside the hash table. Clustering can be avoided with by using quadratic probing instead of traditional linear probing. Hashing is also useful in cryptography. In cryptography, the hash function is modified to be one-way or easy to compute a hash for a value but not easy

Applications

Hashing is not just a simple way to store data in a dictionary like format. Hashing can be used to quickly find elements of text in a document. For example the Karp-Rabin algorithmn hashes each word on a page, when someone searches throught the page the algorithm checks if the search words hash collides wit words in the hashed document.
