Dijksta and Shortest Path

Shortest Path Basics

Most shortest path algorithms realy on a few basic steps. The First step is intiation where the start and end node is given and the distance to every node is initiated. The statrt node is given a distance of 0 and all the other nodes have distances of infinity. THe second or main step involves picking an edge on the graph. This varies from algorithm to algorithmn. Finally a given node is "relaxed" ie if the current distance d[v] is larger then the distance from an adjacent node + the edge weight. the node distance d[v] is replaced with the new distance. This process is then repeated until the distance d[v] is equal to the minimum weight distance.

Dijkstra Summary

Dijkstra algorithmn works remarkebly simialr to the genera description given above. The edges are selected by selecting the smallest weight edge from a given node and going to the adjacent node attached to that edge. This can be seen as a sort of greedy strategy. However, unlike regualer greeedy algorithms dijkstra's gives a globally optimal solution.

Technical Dijkstra

Dijksta's algorithmn assumes that there are no negative weight cycles in a graph. As opposed to Bellman-Ford which finds the shortest paths in any weighted directed graphs Time wise Dijkstra is fairly fast. The Fastest implementation of dijkstra using a fibbonaci heap takes $O(E + V * log(V))$ time where V is the the number of vertices or nodes and E is the number of edges. Alternatives to Dijkstra's include Bellman ford which has V*E complexity but is able to deal with negative edge weights and cycles. Dijkstras algorithmn efficiency does not depend on the edge weights. However , in real life edge weights can be very important. Large edgeweights might cause memory problems. Further, When certain properties are present like bounded edge weights a montone priority queue can improve the running time further.
